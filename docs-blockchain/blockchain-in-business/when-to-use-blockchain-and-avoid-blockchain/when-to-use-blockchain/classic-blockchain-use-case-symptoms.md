# Classic Blockchain Use Case Symptoms

## Classic Blockchain Use Case `Symptoms`

1. **Audit / Review**
   * this process is subject to audit or later review by external stakeholders, regulators, etc
2. **Can't be solved with a database**
   * a problem which cannot be solved by a database due to technical limitations
   * ...when process of elimination applied&#x20;
3. **Non-technical (human) problems / barriers**
   * a problem that can't be solved with conventional technology due to nontechnical reasons such as distribution of power, control & authority&#x20;
4. **Systems integration**
   * to integrate disparate line of business (LoB) systems, oftentimes across organizational boundaries
   * a reliable way to persist state across LoB applications



## Good Use Case Patterns

* ﻿﻿Solution designed for large audience, cross-organizational usage / participation
* ﻿﻿Need for high degree of transparency
* ﻿﻿No single participant should have disproportional authority / power
* ﻿﻿Can benefit from the use of shared IT infrastructure



## Bad Use Case Patterns

* ﻿﻿Solution to be used internally by a single organization
* ﻿﻿High degree of centralized control is desired or beneficial
* Objectives can be achieved using conventional (pre-blockchain) technology
* ﻿﻿Preference for closed-source software, tooling, components
