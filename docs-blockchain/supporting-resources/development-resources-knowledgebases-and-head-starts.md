## Other

#### Alchemy NFT API

{% embed url="https://www.alchemy.com/nft-api" %}

Documentation: [https://www.alchemy.com/docs](https://www.alchemy.com/docs)

* create, distribute, and monitor NFTs (non-fungible tokens) you create on multiple different blockchains&#x20;
* Works across multiple chains (`Ethereum`, `Polygon`, `Flow`, etc...)



#### Thirdweb SDK

{% embed url="https://thirdweb.com/" %}

Documentation: [https://portal.thirdweb.com/](https://portal.thirdweb.com/)&#x20;

* Great SDK for adding blockchain functionality to existing web1/web2 applications&#x20;
* offer many no-code or low-code components (as matter of adding some configurations)
  * e.g: for adding wallet connectivity



#### Moralis API & SDK&#x20;

{% embed url="https://moralis.com/" %}

Documentation: [https://docs.moralis.com/](https://docs.moralis.com/)

* Many common functions provided out of the box with no code required (with only some configurations) !
  * authenticating users
  * sending and fetching/querying  transactions
  * listening to smart contract events
    * this capability is often times used to integrate Blockchains with other legacy or lone of business systems&#x20;
  * pulling user balances
  * interacting with smart contracts&#x20;



#### OpenZeppelin Contracts Library

{% embed url="https://www.openzeppelin.com/" %}

OpenZeppelin: an organization that develops vets[^1], reviews and makes available smart contracts which contain many commonly used or utility functions&#x20;

OpenZeppelin Solidity Contracts Library: [https://www.openzeppelin.com/solidity-contracts](https://www.openzeppelin.com/solidity-contracts)

* offer secure and time-tested pre-written vetted smart contracts to help accelerate your development activities
* ensure common functions are being performed in a standards-compliant way



#### `Consensys` Ethereum Smart Contract Security Knowledgebase

{% embed url="https://consensysdiligence.github.io/smart-contract-best-practices/" %}

* GREAT knowledgebase covering general design philosophy, patterns & anti-patterns, bug bounties, and ✅ **known attack vectors** to watch out for !
* ❗It's worth it to go through this even if you've chosen any other blockchain platform to develop with...



#### Truffle Boxes

{% embed url="https://archive.trufflesuite.com/boxes/" %}

* Project template in a box, perfect for starting new projects and experiments !
* Designed to work with the Truffle command line API
* different boxes that shows different implementation starting points
  * Angular front end with Ethereum Smart Contract backend
  * integrating Ethereum with non blockchain components&#x20;
  * front end development strategies&#x20;



#### Truffle Guides

{% embed url="https://archive.trufflesuite.com/guides/" %}

* Lots of tutorials and walkthroughs of common solution designs
  * minting and tracking of NFTs
  * decentralized streaming in content sharing applications&#x20;
  * smart contract solutions to manage royalty payment &#x20;
  * etc...







[^1]: critical examinations
