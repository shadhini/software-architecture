# Query Languages

## Query Languages

> **Query**
>
> — request for information from a database —&#x20;

> **`Query`/`Database Query` Language**&#x20;
>
> —  predefined and understandable instructions to make requests to a database —&#x20;

#### Query Languages

* **`SQL`**: Structure Query Language
  * most prevalent
* AQL
* CQL
* Datalog
* DMX



## SQL vs NoSQL Databases

> **NoSQL**: Not Only SQL

| SQL                                | NoSQL                                  |
| ---------------------------------- | -------------------------------------- |
| relational                         | non-relational                         |
| use structured, predefined schemas | dynamic schemas for unstructured data  |



## `DBMS`: Database Management Systems

> **`DBMS`: Database Management System**
>
> * hosts the database
> * query languages should be understood by the DBMS as query is made to the DBMS

## Database Queries

> **Database Queries**
>
> typically user enters queries to&#x20;
>
> * either to **query**
> * or perform **`CRUD`** operation: create, read, update & delete data in the database
>
> database queries — **`statements`**  are either
>
> * **`Select queries`**: `SELECT`
>   * request data from a database
> * **`Action`` ``queries`**: `CREATE`, `INSERT`, `UPDATE`
>   * manipulate data in a database
> * mixture of both
> * administrative functions
>   * creating users
>   * modifying permissions
