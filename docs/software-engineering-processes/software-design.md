# Software Design

## System/Software Design

> **Software Design**
>
> transforming requirements into code; software solution

incorporate guidance on

* system functions
* performance
* security&#x20;
* platform characteristics

design communicates

* business rules
* application logic
* application programming interface design
  * how apps talk to each other or communicate with the database&#x20;
* user interfaces
* database design



### Document ...

* structural components
  * structural elements: **modules** & **sub-modules**
    * ✅️ cohesive & loosely coupled modules
      * **cohesive**: all functionally related elements are grouped together
      * **coupling**: communication between different modules
      * **loosely coupled**: modules should be weakly associated so that changes in one component have minimal effect on other components
* behavioural attributes
  * **behavioural models:** explains what a system does / behavior of the system | not how it does it
  * UML diagrams:&#x20;
    * **state transition** diagram
    * **interaction** diagram
